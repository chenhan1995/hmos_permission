# 技术栈规范
## 前端技术
- Flutter: ^3.22.0
  - 用于构建跨平台应用的UI框架
  - 版本必须保持在 3.22.0 或以上兼容版本

## 状态管理
- Riverpod: ^2.6.1
  - 用于应用状态管理
  - 版本必须保持在 2.6.1 或以上兼容版本

## 后端即服务 (BaaS)
- Firebase
  - 用于提供后端服务支持
  - 包括但不限于：
    - 身份验证
    - 数据库
    - 存储
    - 云函数
    - 消息推送

注意事项：
1. 严格遵守版本要求，不得随意更改版本号
2. 如需升级版本，必须经过评估和批准
3. 确保所有依赖包之间的版本兼容性
4. 在使用 Firebase 服务时遵循最佳实践和安全准则
